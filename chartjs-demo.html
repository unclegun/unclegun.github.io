What is Chart.js?

Chart.js is a powerful, flexible, and open-source JavaScript library for creating beautiful, responsive charts. It's built on HTML5 Canvas and provides a simple yet flexible API for developers.

üé® Key Features

  • 8 Chart Types: Line, Bar, Radar, Doughnut, Pie, Polar Area, Bubble, and Scatter
  • Responsive: Charts automatically resize based on container size
  • Animation: Smooth, customizable animations out of the box
  • Interactivity: Tooltips, legends, and click events
  • Accessibility: ARIA labels and keyboard navigation
  • Customizable: Extensive configuration options
  • Lightweight: Small bundle size (~200KB)
  • Framework Agnostic: Works with React, Vue, Angular, and vanilla JS

Chart.js in .NET Core Web Applications

üí° Integration Approach

Chart.js integrates seamlessly with .NET Core through the View layer. Your .NET backend provides data via API endpoints, and Chart.js renders it on the frontend.

Step 1: Create a Web API Controller

// Controllers/ChartDataController.cs using Microsoft.AspNetCore.Mvc; [Route("api/[controller]")] [ApiController] public class ChartDataController : ControllerBase { [HttpGet("sales")] public IActionResult GetSalesData() { var data = new { labels = new[] { "Jan", "Feb", "Mar", "Apr", "May", "Jun" }, datasets = new[] { new { label = "Sales 2024", data = new[] { 65, 59, 80, 81, 56, 55 }, backgroundColor = "rgba(54, 162, 235, 0.2)", borderColor = "rgba(54, 162, 235, 1)", borderWidth = 2 } } }; return Ok(data); } [HttpGet("dashboard")] public async Task<IActionResult> GetDashboardData() { // Fetch from database using Entity Framework var salesData = await _context.Sales .Where(s => s.Date >= DateTime.Now.AddMonths(-6)) .GroupBy(s => s.Date.Month) .Select(g => new { Month = g.Key, Total = g.Sum(s => s.Amount) }) .ToListAsync(); return Ok(salesData); } }

Step 2: Create Razor View with Chart.js

@* Views/Dashboard/Index.cshtml *@ <div class="container"> <h2>Sales Dashboard</h2> <canvas id="salesChart"></canvas> </div> @section Scripts { <script src="https://cdn.jsdelivr.net/npm/chart.js"></script> <script> // Fetch data from API fetch('/api/chartdata/sales') .then(response => response.json()) .then(data => { const ctx = document.getElementById('salesChart').getContext('2d'); new Chart(ctx, { type: 'line', data: data, options: { responsive: true, plugins: { title: { display: true, text: 'Monthly Sales Data' } } } }); }); </script> }

Step 3: Using with Razor Pages (Alternative)

// Pages/Dashboard.cshtml.cs public class DashboardModel : PageModel { private readonly IChartService _chartService; public DashboardModel(IChartService chartService) { _chartService = chartService; } public string ChartDataJson { get; set; } public async Task OnGetAsync() { var chartData = await _chartService.GetSalesDataAsync(); ChartDataJson = JsonSerializer.Serialize(chartData); } } // In the Razor Page: <script> const chartData = @Html.Raw(Model.ChartDataJson); const ctx = document.getElementById('myChart').getContext('2d'); new Chart(ctx, { type: 'bar', data: chartData }); </script>
üéØ .NET Core Best Practices
  • Separate API Layer: Create dedicated API controllers for chart data
  • Use DTOs: Map database entities to Data Transfer Objects
  • Implement Caching: Cache frequently accessed chart data using IMemoryCache
  • Async Operations: Use async/await for database queries
  • Error Handling: Implement proper exception handling and return appropriate status codes
  • Authorization: Secure API endpoints with [Authorize] attributes
  • Validation: Validate query parameters using model validation

Best Design Practices

üé® Visual Design

  • Color Consistency: Use your brand colors for charts
  • Accessibility: Ensure sufficient color contrast (WCAG 2.1)
  • Simplicity: Don't overcrowd charts with too much data
  • Responsive: Charts should adapt to all screen sizes
  • Animations: Use subtle animations for better UX

‚ö° Performance

  • Lazy Loading: Load charts only when visible
  • Data Decimation: Sample large datasets
  • Destroy Charts: Clean up chart instances when not needed
  • Debounce Updates: Limit real-time update frequency
  • Canvas vs SVG: Use Canvas for better performance

üì± Bootstrap Integration

  • Grid System: Use Bootstrap's responsive grid for chart layout
  • Cards: Wrap charts in Bootstrap cards for consistent styling
  • Spacing: Utilize Bootstrap's spacing utilities (mt, mb, p-*)
  • Themes: Leverage Bootstrap's color scheme
  • Responsive Breakpoints: Adjust chart sizes at different breakpoints

‚ôø Accessibility

  • Alt Text: Provide text descriptions for screen readers
  • Keyboard Navigation: Ensure charts are keyboard accessible
  • ARIA Labels: Use appropriate ARIA attributes
  • Color Blind Friendly: Don't rely solely on color
  • Data Tables: Provide tabular data as alternative

// Advanced Configuration with Plugins const config = { type: 'line', data: data, options: { responsive: true, maintainAspectRatio: false, plugins: { title: { display: true, text: 'Custom Chart Title', font: { size: 18 } }, legend: { position: 'bottom', labels: { usePointStyle: true, padding: 15 } }, tooltip: { backgroundColor: 'rgba(0,0,0,0.8)', callbacks: { label: function(context) { return context.dataset.label + ': $' + context.parsed.y; } } } }, scales: { x: { grid: { display: false } }, y: { beginAtZero: true, ticks: { callback: function(value) { return '$' + value; } } } }, animation: { duration: 2000, easing: 'easeInOutQuart' } } };
// Dynamically Update Chart Data function addData(chart, label, data) { chart.data.labels.push(label); chart.data.datasets.forEach((dataset) => { dataset.data.push(data); }); chart.update(); } function removeData(chart) { chart.data.labels.pop(); chart.data.datasets.forEach((dataset) => { dataset.data.pop(); }); chart.update(); } // Real-time data update example setInterval(() => { const newValue = Math.random() * 100; addData(myChart, 'New', newValue); // Keep only last 10 data points if (myChart.data.labels.length > 10) { removeData(myChart); } }, 5000);
